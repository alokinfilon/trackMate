1. 🗄️ The Database Blueprint: gallery.model.jsThis file defines how an uploaded photo entry looks in your MongoDB.
It doesn't hold the physical image, but it links all the tracking details together.
Its Primary Job: Enforcing strict rules on image metadata before it gets saved.Key Fields Enforced Inside:userId: 
The database link to the user who uploaded the photo.tripId (Collection Reference): The ObjectId pointing to the specific trip this
photo belongs to.imageUrl: A string storing the secure web link where the image physically lives (e.g., AWS S3 or Cloudinary path).
caption: A text string for the image description.
accessibility: A strict Enum (['public', 'shared', 'private']) to control who can view the photo.


2. 🎮 The Brains & Logic: gallery.controller.jsThis file contains the functions that handle the heavy lifting: 
processing the incoming binary file, uploading it, and saving the record to the database.Function 1: 
uploadPhoto(req, res)What it does: It intercepts the raw file from the frontend request. It uploads that file to a storage server 
(local folder or cloud), grabs the resulting web URL string, calculates the metadata package, and saves it to your gallery.model.js.
It returns the saved record to the phone interface.Function 2: getTripGallery(req, res)What it does: Reads a tripId from the URL. 
It queries MongoDB to find all images where the tripId matches, while filtering out files based on their accessibility 
rights (e.g., preventing you from seeing another user's "private" photo).


3. 🛣️ The Traffic Cop: gallery.routes.jsThis file connects your web address endpoints to your logic.
 It requires an extra tool called Multer (a middleware) to process binary file uploads before handing them to the controller.
 The Configured Route Paths Inside:POST /api/gallery/:tripId:Executes verifyUserAuth to identify the user.Executes Multer to parse the 
 binary image stream.Forwards the file string data straight to uploadPhoto.GET /api/gallery/:tripId: Listens for gallery load requests 
 and forwards execution straight to getTripGallery.
